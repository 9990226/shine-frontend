## 測試者需要準備的東西（簡單）

每位測試者必須自己準備：
1. **自己的 Gmail App Password**（最重要，唯一需要你自己提供的資訊）

2. **1~3 個職位搜尋來源**（依你的方案：Lv1 最多 1 個、Lv2 最多 2 個、Lv3 最多 3 個）

   - 複製那些「已篩選結果頁」的完整網址，貼到工具裡的三個獨立來源框（每個框一個完整連結）。
   - 之後工具會**自動**幫你瀏覽這些頁面、從每個職缺廣告找出申請電郵、排除重複職缺、準備信件。

寫好一次 Universal Cover Letter + CV 公開連結即可。沒有其他金鑰需要填。

## 如何部署給 3-5 人測試（非技術創辦人步驟）

### 步驟 1: 部署後端（Render 免費版，5 分鐘）

1. 把整個 `shine/backend` 資料夾壓縮上傳到 GitHub 一個新 repo（或直接用現有）。
2. 去 https://render.com 登入（用 GitHub）。
3. New + → Web Service
4. 選擇你的 repo
5. Build Command: `npm install`
6. Start Command: `npm start`
7. 部署完成後，複製你的 Render URL（例如 https://shine-backend-beta.onrender.com）

### 步驟 2: 更新前端（已完成）

前端已預設使用你的 Render 後端：**https://shine-backend-byii.onrender.com**

把整個 `shine/` 資料夾（包含更新後的 index.html + assets）上傳到 Hostinger 的 `public_html/shine` 資料夾（或 scp 到 VPS）。

**重要（後端更新）**：如果你修改了後端程式碼，記得重新部署到 Render 並確保相關 AI 處理的環境變數已設定好（由 2C-AI 支付費用）。前端已完全隱藏後端地址與金鑰輸入。

測試者打開 `https://2c-ai.com/shine` 即可使用。所有技術細節（後端地址、AI 處理金鑰）都已隱藏，用戶只需專注填設定與來源連結。

### 步驟 3: 給測試者使用

傳給他們兩個東西：
- 你的 Hostinger 網址（例如 2c-ai.com/shine）
- 上面「測試者需要準備的東西」的清單

他們打開網址後（portal 隱藏，點「進入我的工具」才顯示）：
1. 填寫 **Universal Cover Letter**（最重要，會 100% 原樣寄出，一字不改）
2. 設定關鍵字、機構黑名單、CV 公開連結
3. 選擇你的方案（決定最多能指定幾個來源網站）
4. 將已篩選好的職位搜尋來源連結分別貼到工具裡的三個獨立來源框（每個框填一個完整結果頁連結）
5. 填 Gmail 電郵 + App Password
6. 按「儲存我的設定」
7. 按「開始自動掃描與準備申請」

工具會自動：
- 根據你指定的來源網站內容找出職缺與申請方式
- 自動排除跨網站重複的同一個職缺
- 準備好預覽（全部用你原封不動的信）
- 讓你確認後，用你的 Gmail 以 30-180 秒隨機自然間隔一批批寄出

你完全不用再自己反覆開網站、找電郵、抄來抄去寄信。手動區只供測試時備用。

整個過程由 2C-AI 支援 AI 處理費用（付費用戶）。介面上看不到任何後端地址或 AI 金鑰欄位。
